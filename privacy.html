Политика приватности

Коротко: мы не собираем персональные данные, не ведём трекинг и не показываем рекламу. Всё хранится локально в вашем браузере.

Коротко о главном

✓ Не собираем персональные данные ✓ Хранение локально в браузере ✓ Open source, MIT ⚠ AI-режим: запросы напрямую к провайдерам ⚠ Баг-репорт: только по клику пользователя

Последнее обновление: 20 апреля 2026

1. Какие данные обрабатывает расширение

NMO Helper работает локально в вашем браузере. На серверы разработчика данные отправляются только в одном случае — при явном нажатии кнопки «Сообщить о проблеме» (см. раздел 3). Расширение сохраняет:

  • Настройки пользователя (выбранные режимы работы, параметры AI)
  • Кеш найденных ответов для ускорения повторных тестов

Всё это хранится локально через API chrome.storage. Разработчик не имеет доступа к этим данным. Вы можете в любой момент очистить их, удалив расширение или очистив хранилище в настройках браузера.

2. К каким внешним сервисам обращается расширение

Запросы отправляются напрямую из вашего браузера, минуя серверы разработчика:

  • База ответов — публичные сайты rosmedicinfo.ru и 24forcare.com
  • AI-режим (опционально) — OpenAI, Anthropic, Google, DeepSeek через ваш собственный API-ключ или российский шлюз proxyapi.ru

AI-режим включается пользователем вручную. Содержимое тестовых вопросов отправляется в выбранный AI-сервис для получения ответа. Ключ API хранится только в локальном хранилище браузера пользователя.

3. Баг-репорты (только по инициативе пользователя)

Если парсер не смог разобрать ответы на странице источника, под статусом появляется кнопка «Сообщить о проблеме». При нажатии открывается превью со списком данных, которые будут отправлены. Отправка происходит только после повторного подтверждения пользователем — никаких автоматических или фоновых отправок не делается.

На сервер nmo-helper.ru/api/bug-report передаются:

  • Название теста (тема) как оно отображается на странице НМО
  • Текст текущего вопроса и его HTML-разметка — для диагностики парсинга
  • Список вариантов ответов с текущей страницы
  • URL страницы-источника (rosmedicinfo.ru или 24forcare.com), на которой искались ответы
  • Версия расширения и версия браузера

Не передаётся: URL страницы НМО, учётные данные, cookies. IP фиксируется на сервере только как счётчик для rate-limit и автоматически удаляется через 24 часа.

На клиенте и на сервере действуют лимиты: минимум 5 минут между отправками, не больше 5 отчётов в сутки с одного IP, дубликаты одного и того же вопроса отбиваются в течение 7 дней. Отчёты попадают в закрытый Telegram-канал разработчика и используются исключительно для диагностики парсеров.

4. Аналитика на сайте nmo-helper.ru

На сайте nmo-helper.ru установлен счётчик Яндекс.Метрики для анализа посещаемости. Метрика собирает обезличенные данные: страницу, откуда пришёл пользователь, устройство, браузер, время на странице. IP-адреса обезличиваются. Эти данные используются только для улучшения сайта.

В самом расширении NMO Helper никакой аналитики, телеметрии или счётчиков нет.

5. Cookies

Сайт nmo-helper.ru использует только технические cookies Яндекс.Метрики для корректной работы аналитики. Рекламные и маркетинговые cookies не используются.

6. Передача данных третьим лицам

Разработчик не передаёт и не продаёт никакие данные третьим лицам. Данные, которые отправляются во внешние сервисы (базы ответов, AI-провайдеры), регулируются политиками приватности этих сервисов — подключение к ним происходит напрямую из браузера пользователя.

7. Права пользователя

Вы можете в любой момент:

  • Удалить расширение — это удалит все связанные с ним локальные данные
  • Очистить кеш ответов через настройки расширения
  • Отключить AI-режим в настройках
  • Не нажимать кнопку «Сообщить о проблеме» — без явного клика данные на сервер не уходят
  • Заблокировать Яндекс.Метрику на сайте через настройки браузера или расширения-блокировщика

8. Исходный код и прозрачность

NMO Helper — полностью open source проект под лицензией MIT. Вы можете проверить, как работает расширение, изучив исходный код на GitHub. Сборка проходит проверку VirusTotal и публикуется в официальных магазинах расширений (Firefox Add-ons, Opera Add-ons).

9. Контакты

Вопросы и предложения по обработке данных — через форму обратной связи или через issues на GitHub.

10. Изменения в политике

При существенных изменениях этого документа дата в начале страницы будет обновлена. Рекомендуем периодически проверять её.
